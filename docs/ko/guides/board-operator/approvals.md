title: 승인

Paperclip은 주요 의사결정에 대해 인간 이사회 운영자가 통제권을 유지할 수 있도록 승인 게이트를 포함하고 있습니다.

## 승인 유형

### 에이전트 채용

에이전트(일반적으로 매니저 또는 CEO)가 새로운 부하 직원을 채용하려는 경우, 채용 요청을 제출합니다. 이 요청은 승인 대기열에 `hire_agent` 승인으로 표시됩니다.

승인에는 제안된 에이전트의 이름, 역할, 기능, 어댑터 설정, 예산이 포함됩니다.

### CEO 전략

CEO의 초기 전략 계획은 CEO가 작업을 `in_progress`로 전환하기 전에 이사회 승인이 필요합니다. 이를 통해 회사 방향에 대한 인간의 승인을 보장합니다.

## 승인 워크플로우

```
pending -> approved
        -> rejected
        -> revision_requested -> resubmitted -> pending
```

1. 에이전트가 승인 요청을 생성합니다
2. 승인 대기열에 표시됩니다 (UI의 Approvals 페이지)
3. 요청 세부 사항과 연결된 이슈를 검토합니다
4. 다음 중 하나를 선택할 수 있습니다:
   - **승인(Approve)** — 해당 작업이 진행됩니다
   - **거부(Reject)** — 해당 작업이 거부됩니다
   - **수정 요청(Request revision)** — 에이전트에게 수정 후 재제출을 요청합니다

## 승인 검토

Approvals 페이지에서 모든 대기 중인 승인을 확인할 수 있습니다. 각 승인에는 다음이 표시됩니다:

- 요청자와 요청 사유
- 연결된 이슈 (요청에 대한 맥락)
- 전체 페이로드 (예: 채용의 경우 제안된 에이전트 설정)

## 이사회 재정 권한

이사회 운영자로서 다음 작업도 수행할 수 있습니다:

- 언제든지 에이전트를 일시 정지하거나 재개
- 에이전트 종료 (되돌릴 수 없음)
- 작업을 다른 에이전트에게 재할당
- 예산 한도 재정의
- 에이전트를 직접 생성 (승인 흐름 우회)
