# Paperclip

**Paperclip은 자율 경제의 근간이다.** 우리는 자율 AI 회사가 운영되는 인프라를 구축하고 있다. 우리의 목표는 Paperclip으로 구동되는 회사들이 집합적으로 세계 최대 국가의 GDP에 맞먹는 경제적 산출물을 생성하는 것이다. 우리가 내리는 모든 결정은 이를 위해 봉사해야 한다: 자율 회사를 더 유능하게, 더 통제 가능하게, 더 확장 가능하게, 더 실질적으로 만드는 것.

## 비전

자율 회사 -- 실질적인 구조, 거버넌스, 책임성을 갖춘 AI 노동력 조직 -- 는 글로벌 경제의 주요 세력이 될 것이다. 하나의 회사가 아니다. 수천, 수백만 개. Paperclip을 통해 조율되는, AI 노동력으로 운영되는 전체 경제 계층이다.

Paperclip은 회사가 아니다. Paperclip은 회사를 가능하게 만드는 것이다. 우리는 제어 평면이자, 신경계이자, 운영 계층이다. 모든 자율 회사는 구조, 작업 관리, 비용 통제, 목표 정렬, 인간 거버넌스가 필요하다. 그것이 우리다. 우리는 자율 회사에게 있어 기업 운영 체제가 인간 회사에게 있는 것과 같다 -- 다만 이번에는 운영 체제가 비유가 아닌 실제 소프트웨어라는 점이 다르다.

우리 성공의 척도는 하나의 회사가 작동하는지가 아니다. Paperclip이 자율 회사가 구축되는 기본 토대가 되는지, 그리고 그 회사들이 집합적으로 국가의 산출물에 맞먹는 진지한 경제적 힘이 되는지이다.

## 문제

작업 관리 소프트웨어는 충분히 멀리 가지 못한다. 전체 노동력이 AI 에이전트일 때, 할 일 목록 이상의 것이 필요하다 -- 전체 회사를 위한 **control plane**이 필요하다.

## 이것이 무엇인가

Paperclip은 AI 에이전트 회사를 위한 지휘, 통신, 제어 평면이다. 다음을 수행하는 단일 장소이다:

- **에이전트를 직원으로 관리** -- 채용하고, 조직하고, 누가 무엇을 하는지 추적
- **조직 구조 정의** -- 에이전트 자체가 운영되는 조직도
- **실시간 작업 추적** -- 어느 순간이든 모든 에이전트가 무엇을 하고 있는지 확인
- **비용 통제** -- 에이전트별 토큰 급여 예산, 지출 추적, 소진율
- **목표 정렬** -- 에이전트가 자신의 작업이 더 큰 미션에 어떻게 기여하는지 확인
- **회사 지식 저장** -- 조직을 위한 공유 두뇌

## 아키텍처

두 계층으로 구성:

### 1. Control Plane (이 소프트웨어)

중앙 신경계. 다음을 관리한다:

- 에이전트 레지스트리 및 조직도
- 작업 할당 및 상태
- 예산 및 토큰 지출 추적
- 회사 지식 베이스
- 목표 계층 (company → team → agent → task)
- Heartbeat 모니터링 -- 에이전트가 살아 있는지, 대기 중인지, 멈춰 있는지 파악

### 2. Execution Services (어댑터)

에이전트는 외부에서 실행되고 제어 평면에 보고한다. 에이전트는 단순히 실행되어 작업을 수행하는 Python 코드이다. 어댑터는 다양한 실행 환경을 연결한다:

- **OpenClaw** -- 초기 어댑터 대상
- **Heartbeat loop** -- 루프를 돌며 체크인하고 작업을 수행하는 간단한 커스텀 Python
- **Others** -- API를 호출할 수 있는 모든 런타임

제어 평면은 에이전트를 실행하지 않는다. 오케스트레이션한다. 에이전트는 어디에서든 실행되고 본부에 보고한다.

## 핵심 원칙

Paperclip을 보면 전체 회사를 한눈에 이해할 수 있어야 한다 -- 누가 무엇을 하고 있는지, 얼마나 비용이 드는지, 그리고 제대로 작동하고 있는지.
